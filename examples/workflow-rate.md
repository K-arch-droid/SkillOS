# 示例：评分审查一个 Skill

## 场景

你写了一个叫 `auto-email` 的 Skill，想检查质量。

## 命令

```bash
python skillos.py rate ./auto-email
```

## 输出示例

```
# Skill 审查报告：auto-email

**检测类型：** methodology
**综合等级：** C (72/100)

## 评分明细

| 维度             | 得分 | 权重 | 加权 |
|------------------|------|------|------|
| Description 质量 | 60   | 25   | 15.0 |
| Frontmatter 有效性 | 85 | 20   | 17.0 |
| 长度与披露       | 100  | 15   | 15.0 |
| 结构匹配度       | 80   | 15   | 12.0 |
| 示例质量         | 40   | 10   |  4.0 |
| 简洁性           | 100  | 10   | 10.0 |
| 反模式规避       | 80   |  5   |  4.0 |

## 优势
- 正文长度合理（120 行）

## 发现（按优先级）

### P1 — Description 缺少指令式触发语态
**原因：** 被动描述降低激活可靠性
**修复：** 添加 'Use this skill whenever the user wants to…'

### P1 — 只有 1 个引号触发短语（需要 ≥3）
**原因：** 触发短语不足导致激活不稳定
**修复：** 添加 ≥3 个用户会说的自然语言短语

### P1 — 缺少反模式示例
**原因：** 反模式帮助避免常见错误
**修复：** 添加 ❌ 反模式示例

## 修复 P0+P1 后预估等级：A (95/100)
```

## 下一步

根据报告中的修复建议修改 SKILL.md，然后重新运行 `rate` 验证。
